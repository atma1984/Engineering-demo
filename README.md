# EngineeringSystem

**EngineeringSystem** — корпоративная информационная система для сопровождения
полного жизненного цикла работы с заказчиком:
от первичного запроса и фиксации потребности
до инженерного проекта, калькуляции, коммерческого предложения и договора.

Система проектируется как **единый продукт** с:
- клиентским приложением на **Windows Forms**,
- backend-сервером на **ASP.NET Core Web API**,
- единой доменной моделью, отражающей реальные бизнес-процессы предприятия.

---

## Назначение системы

EngineeringSystem предназначена для:
- централизованного ведения заказчиков, контактов и организационной структуры;
- фиксации и анализа первичных запросов клиентов;
- управления инженерными проектами и их стадиями;
- работы с номенклатурой, исполнениями изделий и BOM;
- хранения и анализа цен (внутренних и рыночных);
- формирования коммерческих предложений и договоров;
- аналитики и отчётности по всем этапам.

---

## Общая архитектура

Система строится по следующим архитектурным принципам:

- **Monorepo** (единый репозиторий)
- **Clean Architecture**
- **Modular Monolith**
- Чёткое разделение ответственности между слоями

### Технологический стек

- **Client**: Windows Forms (.NET), паттерн MVP
- **Backend**: ASP.NET Core Web API
- **База данных**: Microsoft SQL Server
- **Транспорт**: HTTP / REST API

---

## Архитектурные слои

### Client
- Отвечает только за UI и пользовательские сценарии
- Не содержит бизнес-логики
- Взаимодействует с backend исключительно по HTTP

### API
- Точка входа в систему
- Принимает HTTP-запросы
- Выполняет валидацию, авторизацию и маршрутизацию
- Не содержит бизнес-логики

### Application
- Реализует use-case’ы системы
- Управляет бизнес-процессами и транзакциями
- Оркестрирует доменные объекты
- Не зависит от UI и инфраструктуры

### Domain
- Содержит бизнес-смысл системы
- Описывает сущности, агрегаты, value object’ы и правила
- Не зависит от технологий
- Самый стабильный слой системы

### Infrastructure
- Реализует работу с БД (EF Core, SQL Server)
- Внешние сервисы
- Технические детали хранения и интеграций
- Не принимает бизнес-решений

---

## Основные домены системы

- **Customers**  
  Заказчики, контакты, подразделения, организационная структура, география

- **Orders**  
  Первичный запрос клиента, фиксация потребности, участников и неопределённостей

- **Projects**  
  Инженерный и коммерческий проект, стадии, управление жизненным циклом

- **Nomenclature**  
  Классификатор изделий, номенклатура, исполнения, BOM

- **Pricing**  
  Ценообразование, типы цен, история цен, аналитика

- **Documents**  
  Документы как управляемые бизнес-объекты (версии, статусы)

- **Communications**  
  Письма и коммуникации (входящие и исходящие), связанные с заказами и проектами
